# Security policy

## Supported versions

brain-cli is a point-in-time public snapshot released as a companion to a
written article. It is not under active security maintenance. The `v0.1.0`
tag is the only version.

## Reporting a vulnerability

If you find a security issue, please open a private report through GitHub's
security advisory workflow:

1. Go to the repository's **Security** tab on GitHub.
2. Click **Report a vulnerability**.
3. Fill in the advisory form with a short description, reproduction steps,
   and any affected file paths or commit hashes.

GitHub will route the report privately. Please do not open a public issue
for security problems.

## No bounty program

There is no bug bounty program for brain-cli. Reports are accepted on a
goodwill basis and responded to when possible. No payment is offered.

## Scope

In scope: the Rust code in this repository at its published commit.

Out of scope: anything in a fork, anything in the companion `puck-playbook`
repository (report those there), and anything in the orchestration article's
website.
